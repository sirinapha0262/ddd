🤖 ข้อ 1: ASM Chart – เครื่องซักผ้าหยอดเหรียญ

📖 ASM Chart คืออะไร?

ASM (Algorithmic State Machine) Chart คือ แผนภาพที่แสดงการทำงานของระบบดิจิทัลแบบ Sequential โดยมีองค์ประกอบหลัก 3 ส่วน:

⬜ State Box
รูปสี่เหลี่ยม = สถานะปัจจุบัน เช่น IDLE, WASHING
◇ Decision Box
รูปเพชร = เงื่อนไข เช่น Coin_10 = 1?
⬭ Output Box
รูปวงรี = ผลลัพธ์ เช่น Wash=1, Dry=1

🧺 โจทย์: เครื่องซักผ้าหยอดเหรียญ

📋 เงื่อนไขของระบบ
  • 1️⃣ Input: Coin_10 — รับเหรียญ 10 บาท จนครบ 40 บาท
  • 2️⃣ Output: Wash — เริ่มซัก 1 รอบ หลังครบ 40 บาท
  • 3️⃣ Input: Timer_Done — สัญญาณหลังซักเสร็จ → ทำการปั่นแห้ง
  • 4️⃣ Output: Dry — ปั่นแห้ง จนเสร็จสิ้น
  • 5️⃣ กลับสู่สถานะรอรับเหรียญ
สถานะทั้งหมด: S0 (IDLE) → S1 (10฿) → S2 (20฿) → S3 (30฿) → S4 (WASH) → S5 (DRY) → S0

📐 สูตร: [(p ∨ q) ∧ ¬p] → q

🔍 วิธีอ่านสูตร

ถ้า (p หรือ q เป็นจริง) และ (p เป็นเท็จ) แล้ว q ต้องเป็นจริง

pqp ∨ q¬p(p∨q) ∧ ¬p[(p∨q)∧¬p] → q
TT TFF T
TF TFF T
FT TTT T
FF FTF T
✅ ผลสรุป: เป็นสัจนิรันดร์ (Tautology)!
คอลัมน์สุดท้ายเป็น T ทุกแถว → สูตรนี้เป็นจริงเสมอ ไม่ว่า p, q จะเป็นอะไร

🎮 ลองกด p และ q เอง!

p ∨ q = False

❓ สัจนิรันดร์ (Tautology) คืออะไร?

📖 นิยาม
สูตรที่เป็น จริงทุกกรณี ไม่ว่าจะแทนค่าตัวแปรอย่างไร เรียกว่า "สัจนิรันดร์"
✅ ตัวอย่าง Tautology
  • p ∨ ¬p (เป็นจริงเสมอ)
  • p → p
  • [(p∨q)∧¬p] → q ✓
❌ ไม่ใช่ Tautology
  • p ∧ q (อาจเป็น F)
  • p → q (อาจเป็น F)
  • p ∨ q (เป็น F เมื่อ p=F,q=F)
🔄 ข้อ 3-4: กฎสมมูล & อาร์กิวเมนต์

📝 พิสูจน์: ¬(p → q) ≡ p ∧ ¬q

🎯 เป้าหมาย
ต้องแปลงฝั่งซ้าย ¬(p → q) ให้ได้ p ∧ ¬q โดยใช้กฎสมมูล
¬(p → q)
เริ่มต้น
≡ ¬(¬p ∨ q)
Implication Law: p→q ≡ ¬p∨q
≡ ¬(¬p) ∧ ¬q
De Morgan's Law: ¬(A∨B) ≡ ¬A∧¬B
≡ p ∧ ¬q
Double Negation: ¬(¬p) ≡ p ✅
✅ พิสูจน์สำเร็จ!
ใช้กฎ 3 ขั้นตอน: Implication → De Morgan → Double Negation

🔍 ข้อ 4: ตรวจอาร์กิวเมนต์ p→q, ¬q ⊢ ¬p

📋 อาร์กิวเมนต์ที่ให้
  • สมมติฐาน 1: p → q
  • สมมติฐาน 2: ¬q
  • สรุป: ¬p
สมมติฐาน: p → q
กำหนดให้
ทราบว่า: ¬q
กำหนดให้
∴ ¬p
Modus Tollens (กฎอ้างเหตุผลแบบปฏิเสธ)
✅ อาร์กิวเมนต์สมเหตุสมผล!
ใช้กฎ Modus Tollens: ถ้า p→q และ ¬q แล้ว ¬p เสมอ

📊 ตรวจด้วยตารางความจริง

pqp→q¬q¬pสมมติฐานจริงทั้งคู่?สรุปจริงไหม?
TTTFFF (¬q เท็จ)ไม่นับ
TFFTFF (p→q เท็จ)ไม่นับ
FTTFTF (¬q เท็จ)ไม่นับ
FFTTT✓ ทั้งคู่จริง✓ จริง
กรณีที่สมมติฐานจริงทั้งคู่ (แถวสุดท้าย) → สรุปเป็นจริงด้วย → สมเหตุสมผล ✓

📚 กฎสมมูลที่ต้องจำ

1️⃣ Implication Law
p → q ≡ ¬p ∨ q
2️⃣ De Morgan's Laws
¬(p∧q) ≡ ¬p∨¬q
¬(p∨q) ≡ ¬p∧¬q
3️⃣ Double Negation
¬(¬p) ≡ p
4️⃣ Modus Ponens
p→q, p ⊢ q
5️⃣ Modus Tollens
p→q, ¬q ⊢ ¬p
6️⃣ Contrapositive
p→q ≡ ¬q→¬p

✅ ข้อ 6: Tautology Checker

🐛 โค้ดที่มีบั๊ก
def is_tautology(): results = [] for p in [True, False]: for q in [True, False]: lhs = p and ((not p) or q) # ❌ lhs = p AND (p→q) expression = (not lhs) or q # ❌ expression = lhs → q (ผิดสูตร) results.append(expression) return all(results) print(f"Is it a tautology? {is_tautology()}")
🐛 ปัญหาที่พบ

โค้ดต้องการตรวจสูตร [(p∨q)∧¬p] → q แต่โค้ดเดิมคำนวณสูตรผิด

  • lhs ควรเป็น (p or q) and (not p) ไม่ใช่ p and ((not p) or q)
  • expression ควรเป็น (not lhs) or q ซึ่งถูกแล้ว แต่ lhs ผิด
✅ โค้ดที่ถูกต้อง
def is_tautology(): results = [] for p in [True, False]: for q in [True, False]: lhs = (p or q) and (not p) # ✅ (p∨q) ∧ ¬p expression = (not lhs) or q # ✅ lhs → q results.append(expression) return all(results) print(f"Is it a tautology? {is_tautology()}") # Output: Is it a tautology? True
🔍 อธิบาย

(not lhs) or q คือการคำนวณ lhs→q เพราะ p→q ≡ ¬p∨q

all(results) จะคืนค่า True ก็ต่อเมื่อทุกค่าใน list เป็น True = สูตรเป็น Tautology

🔢 ข้อ 7: Basic Proof Logic – Prime Numbers

🎯 โจทย์ตรรกะที่ทดสอบ

(Prime(n) AND n > 2) → IsOdd(n)

"ถ้า n เป็นจำนวนเฉพาะและ n > 2 แล้ว n ต้องเป็นเลขคี่"

def is_prime(n): if n < 2: return False for i in range(2, int(n**0.5) + 1): if n % i == 0: return False return True # ตรวจสอบตรรกะ: (Prime(n) AND n > 2) -> IsOdd(n) found_counterexample = False for n in range(1, 101): condition = is_prime(n) and n > 2 conclusion = n % 2 != 0 # IsOdd(n) # ตรรกะ: NOT condition OR conclusion if condition and not conclusion: # พบ counterexample! print(f"Counterexample found: {n}") found_counterexample = True break if not found_counterexample: print("Logic holds for n=1 to 100") # Output: Logic holds for n=1 to 100
🔍 อธิบายทีละส่วน
โค้ดความหมาย
int(n**0.5)+1ตรวจตัวหารแค่ถึง √n (ประหยัดเวลา)
condition = is_prime(n) and n>2สมมติฐาน: n เป็นเฉพาะ และ >2
conclusion = n%2!=0ข้อสรุป: n เป็นเลขคี่
if condition and not conclusionถ้าสมมติฐานจริงแต่สรุปผิด = counterexample
จำนวนเฉพาะที่ >2 ทุกตัวเป็นเลขคี่ (2 เป็นจำนวนเฉพาะคู่เดียว) → Logic holds ✓
